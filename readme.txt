Projekt – wymagania
1. Stworzenie prostej aplikacji banku wykorzystującej co najmniej:
a) HTML
b) CSS
c) PHP
d) Bazę danych MySQL, MariaDB lub SQL Server
e) Wykorzystanie JavaScript mile widziane
Można wykonać projekt wykorzystując framework Laravel
2. Prace będą tworzone w grupach 2/3-osobowych.
3. Oddanie pracy będzie polegało na przesłaniu projektu poprzez Moodle:
a) krótkiego opisu jak działa aplikacja - funkcjonalności (kilka zrzutów ekranu z krótkim komentarzem)
b) plików źródłowych (cały katalog projektu)
c) eksportu struktury bazy danych wraz z przykładowymi danymi
d) krótkiej instrukcji jak uruchomić aplikację. Rekomenduję by działającą aplikację udostępnić
również za pomocą darmowego hostingu.
Moodle:
https://moodle2.e-wsb.pl/mod/assign/view.php?id=2159445
4. Co będzie ocenianie:
a) Funkcjonalność: jako minimum aplikacja musi być w stanie odczytać, zapisać i zaktualizować dane
w bazie (55 punktów):
 dodawanie nowych klientów banku i zarządzanie istniejącymi,
 uwierzytelnianie klientów w aplikacji
 przelewy między klientami banku (termin wykonania przelewu),
 historia operacji,
 wyświetlanie nazwy banku do którego realizowany jest przelew,
 walidacja danych
 wyświetlanie salda klienta (rachunek, karta kredytowa, kredyt)
b) poprawność kodu HTML i CSS - walidacja (5 punktów)
c) responsywność - poprawne wyświetlanie w różnych rozdzielczościach (5 punktów)
d) estetyka (5 punktów)
e) jakość kodu - (10 punktów)
f) dokumentacja w formie komentarzy w kodzie oraz krótkiego opisu jak działa aplikacja - ze zrzutami
ekranu (10 punktów)
g) użyteczność - intuicyjna nawigacja (5 punktów)
h) testowanie - zrozumiałe komunikaty błędów (5 punktów)
5. Ostateczny termin oddania pracy: niedziela, 14 czerwca 2020
6. Warunkiem zaliczenia przedmiotu jest uzyskanie minimum 51pkt.
